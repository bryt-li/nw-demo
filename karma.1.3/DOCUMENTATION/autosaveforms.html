Autosaveforms

This plugins allows you to auto save every xxxx seconds the content of the form into the localstorage so that if something happens to the page before the form has been send it will restore all the entered values. You can set an save time(in seconds) which is used to save the data every xxxx seconds and a expire after time(minutes) which means that the data will only be available for xxxx minutes. You can delete the data after it's been submitted by using a public method.

Files To Include

Files that you have to include in your <head> section.

<script src="js/plugins/autosaveforms.1.0.min.js"></script>

How To Use

How to use HTML elements.

<form data-asf-time="10" data-asf-expireafter="1" id="autosavethisform">
    <input name="name"/>
    <input name="email"/>
    <textarea name="message"></textarea>
    <button type="submit">Save</button>
</form>

Plugin Usage

How to setup and use the plugin.

//init
$('#autosavethisform').autoSaveForms({
    saveTime: 500,
    expireAfter: 1,
    onSave: function(ui){
		//ui.item(element)
        //ui.output(string)
    }
});

//public methods
$('#autosavethisform').autoSaveForms('destroy');
$('#autosavethisform').autoSaveForms('clear', 'autosavethisform');

Plugin Options

A list of plugin options.

Options Type Default Desctription Since
saveTime integer 500 The interval save time in seconds. 1.0
expireAfter integer 1 Set an expired time that if the user doens't return to the form that the data is't kept stored forever i the localstorage. This is set in days not seconds! 1.0
onSave function function(ui){} Callback function that runs if the form gets saved. 1.0

Plugin Callback Functions

Below a list of callback function parameters.

Function Parameter Returns Desctription Since
onSave ui.item element This is the form that is beeing used. 1.0
ui.output string Returns a string of the saved values. 1.0

Usage Data Atributes

Data atrributes are used by some of our custom plugins to set/get data.

Dataset Element Example Desctription Since
data-asf-time form The interval of the auto saving. This is in seconds. 1.0
data-asf-expireafter form Keep the saved data for a certain period, after this period the data will be cleared from the localstorage. This is in days. 1.0

Public Methods

A list of public methods.

Method Parameter Desctription Since
destroy no Removes the plugin functionality completely. This will return the element back to its pre-init state. 1.0
clear id of the form Removes the localstorage data. 1.0

Globally And Per-Call Overridable Options

This plugin has per-call(as the example above shows) and globally overridable options. Globally options can be used to override the default options, this is best used if you want to override the plugin options by default, this will prevent any issues with the plugin, as you dont have to change any thing in the plugin core it self.

How to use: $.fn.pluginName.key = 'value';
